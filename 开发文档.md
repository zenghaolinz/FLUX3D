# Neural Mesh Generator 开发文档

**基于深度学习的三维网格生成系统**



**适用平台**: Windows 10/11 64-bit

---

## 1. 系统架构概述

Neural Mesh Generator 采用前后端分离的客户端-服务器（C/S）架构。前端负责图形用户界面（GUI）展示、参数组装与状态管理；后端基于 **ComfyUI** 推理框架，通过节点化工作流处理底层的张量计算与模型推理。

- **前端技术栈**: Python 桌面级 GUI（启动入口 `launcher.py`）。

- **通信协议**: HTTP REST API（提交生成任务、获取结果） + WebSocket（监听实时进度和状态）。

- **后端推理引擎**: ComfyUI (基于 PyTorch 与 CUDA 12.x)。

- **核心生成模型**:
  
  - 2D 生成/风格融合：**FLUX.2** (包含 4B 快速模型与 9B 高质量模型)
  
  - 3D 网格生成：**Hunyuan3D-2**

## 2. 开发环境与依赖

## 2.1 硬件与系统要求

为确保在开发和调试过程中模型能够正常加载与推理，建议满足以下环境：

- **OS**: Windows 10/11 64-bit

- **CPU**: Intel i7 / AMD Ryzen 7 及以上

- **RAM**: 32 GB 及以上

- **GPU**: NVIDIA RTX 30/40 系列，推荐 16GB 及以上显存 (最低要求 8GB，如 RTX 3060)

- **存储**: NVMe SSD (至少 100 GB 可用空间用于模型权重和缓存)

## 2.2 开发基础环境

- **Python**: 3.10+

- **CUDA Toolkit**: 12.0+ (推荐 12.6，需与系统 NVIDIA 驱动匹配)

- **C++ 编译环境**: Visual Studio C++ Redistributable 2022 (部分依赖包需要源码编译)

## 4.2 前后端通信机制 (`frontend/api_client.py`)

前端与 ComfyUI 后端的交互主要围绕 `api_client.py` 展开。该模块需实现以下核心逻辑：

1. **工作流组装**: 读取 `workflows/` 下的 JSON 模板，根据前端 UI 状态（文生 3D、图生 3D、双图融合、4B/9B 模式）动态替换占位符（如 Prompt、Seed、Image Path）。

2. **任务提交**: 发送 POST 请求至后端的 `/prompt` 接口提交 JSON 工作流。

3. **状态监听**: 建立 WebSocket 连接监听 `/ws?clientId={id}`，捕获 `execution_start`、`progress` 和 `executed` 事件，并回调给前端更新进度条。

4. **结果获取**: 任务完成后，通过 `/history/{prompt_id}` 接口获取输出节点信息，并下载生成的 `GLB` 文件和相关贴图 (UV, Normal)。

## 5. 核心生成管线 (Pipelines)

系统包含三种核心业务流，均通过调度不同的 ComfyUI 工作流实现：

- **管线 A：文生 3D (Text-to-3D)**
  
  - `输入` -> `Prompt 预处理` -> `FLUX.2 (4B/9B)` -> `生成 2D 参考图 (多视角)` -> `Hunyuan3D-2 注入` -> `GLB 输出`。

- **管线 B：图生 3D (Image-to-3D)**
  
  - `输入图片` -> `背景去除 (Rembg 节点)` -> `图像预处理 (缩放/裁剪至 512x512+)` -> `Hunyuan3D-2` -> `GLB 输出`。

- **管线 C：双图融合 (Dual-Image Fusion to 3D)**
  
  - `图片1 + 图片2 + Prompt` -> `FLUX.2 + IP-Adapter/ControlNet 控制流` -> `风格融合 2D 图` -> `Hunyuan3D-2` -> `GLB 输出`。

## 6. 调试与排错指南

## 6.1 常见开发问题

- **WebSocket 断连或无响应**:
  
  - *排查*: 检查 `api_client.py` 中配置的 ComfyUI 服务端口（默认通常为 8188）是否被占用。确保 `run_nvidia_gpu.bat` 已成功拉起服务。

- **CUDA Out of Memory (OOM)**:
  
  - *排查*: Hunyuan3D 和 FLUX.2 9B 显存消耗极大。开发阶段建议默认将 `质量模式` 锁在 `4B 模型`，并在 ComfyUI 启动参数中加入 `--lowvram` 或 `--medvram` 标志进行调试。

- **模型/节点缺失错误 (Missing Nodes)**:
  
  - *排查*: 检查 ComfyUI 的 `custom_nodes` 目录下是否缺少必要的扩展插件（例如 GGUF 加载节点、3D 预览节点）。检查控制台输出的依赖缺失警告。

## 6.2 日志系统

- **前端日志**: 默认输出至控制台，包含 API 请求负载、耗时及 WebSocket 原始消息。可通过快捷键 `Ctrl+L` 呼出日志窗口。

- **后端日志**: 需查看 `backend/` 目录下启动控制台的打印信息，重点关注张量维度报错 (Shape Mismatch) 或显存溢出 (CUDA Error)。

## 7. 安装包构建

开发者可通过以下步骤生成安装包：

## 7.1 构建环境要求

- Python 3.10+ （已安装 PyInstaller）
- Visual Studio 2022（MSVC 编译器，用于 Nuitka）
- 约 10GB 临时磁盘空间

## 7.2 构建步骤

1. **安装打包工具**:
   
   ```
   pip install pyinstaller
   ```

2. **编译启动器**:
   
   ```
   pyinstaller --onefile --name=AI3DLauncher --console launcher.py
   ```

3. **编译安装程序**:
   
   ```
   pyinstaller --onefile --name=setup --windowed installer/installer_gui.py
   ```

4. **打包所有文件**:
   
   ```
   python installer/create_final.py
   ```

5. **生成产物**: `installer/AI3DModeling_Final_Setup.zip`

## 7.3 安装包内容

| 组件 | 大小 | 说明 |
|------|------|------|
| python_embeded | ~100MB | 便携版 Python 3.12 |
| ComfyUI 核心 | ~300MB | AI 图像生成引擎 |
| custom_nodes | ~20MB | 5 个自定义节点 |
| setup.exe | ~11MB | GUI 安装程序 |
| AI3DLauncher.exe | ~8MB | 启动器 |

**总大小**: 约 5.8GB（压缩后）
